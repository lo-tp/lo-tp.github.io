将vagrant中的本地环境与github pages绑定

By Lotp

Dec 26 2014 Updated:Dec 26 2014

本文延续上一篇使用vagrant搭建hexo本地环境继续讲如何将已经搭建好的hexo与github-pages绑定起来

本文假设你已经拥有了github帐号，并且申请好了github-pages相关项目

获得Github的ssh访问权限

参照这篇教程,使物理机获得免输密码访问github账户的权限

更改虚拟机配置使虚拟机能共享使用物理机的ssh key

编辑Vagrantfile添加config.ssh.forward_agent = true到末尾:

VAGRANTFILE_API_VERSION = "2"
Vagrant.configure(VAGRANTFILE_API_VERSION) do |config|
config.vm.box = "ubuntu14"
config.vm.provision :shell, :inline => "echo \"Asia/Shanghai\" | sudo tee /etc/timezone && dpkg-reconfigure --frontend noninteractive tzdata"
config.vm.provision :shell, path: "initialize.sh"
config.vm.synced_folder "src/", "/home/vagrant/src/",create:true
config.vm.network "private_network", ip: "192.168.50.7"
config.ssh.forward_agent = true
end

更改hexo的配置绑定github-pages

编辑_config.yml文件
更改此文件的最后几行Deployment相关内容

# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:                                                                         
  type: github
  repository: git@github.com:lo-tp/lo-tp.github.io.git

将lo-tp替换成你的github ID

验证账户并定是否成功

在虚拟机下的~/src/目录下执行hexo deploy

成功部署以后，你的hexo博客就已经成功部署到github pages上边了

打开浏览器输入:yourGithubID.github.io

尽情在你的blog上创作吧

PS：部署的过程有一定延时，请耐心等待30分钟左右

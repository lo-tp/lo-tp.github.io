Learning Vim Again

  • Thu 02 April 2015
  • ·
  • Vim

Having been using VIM for several years, I can still find some new VIM tricks. So in order to further improve my editing effiency, I decide to learn Vim again.

I’m reading a book named <Practical VIM>,so this series could either be seen as a record of the process of my learning VIM again or reading notes of that book. Any way, I would feel more than satisfied if these posts could inspire you somehow.

Now let’s get down to business.

Fisrt, I would like to put some commands of VIM here, these commands are conbination of motion and editing, they are uncommon but useful.

Depending on which way is better I’ll explain some of these commands to you with descriptions of their function, and the others with a list of commands of equlvalent function.

  1. ^:Go to the first non-blank character of the line
  2. A:$a
  3. C:c$
  4. s:cl
  5. S:^C
  6. I:^i

Of all the commands above, python programmers will find those connected with ^ extremely useful.

Next, I want to talk about search command in VIM. I’ve been told that I could do a search with f{target} for a long time but I don’t know I can also use F{target} for a reverse search. By the way if you accidentally skip the target and go to the next match, you can use , to make up. One last thing to say: Both f and F can only be used to search target in one line. Talking search, I think * is another powerful technique we should master. With *, you can search the word under cursor in the whole file.

Besides f{target}, there is another search command which will Forward to the character before the next occurrence of{target}. It's t{target}.

Image you want to change I’ve been expecting you, Mister Bond. to I’ve been expecting you. Using f{target}, the minimal keystrokes aref,dfd, with t{target}, you only need to type f,dt. instead.

Don't look down your nose at the little one keystroke we successfully reduced, what if the original sentence is I’ve been expecting you, David Dahood.

If you ask me what do I like most about VIM, I would definitely say it’s the Text Objects mechanism. Basically, it’s something with which you can select and edit elements of the file more effectively and conveniently. It’s difficult to give you an accurate definition of what it is, and instead of that, I’ll cite one paragraph from <Practical VIM>.

Text objects allow us to interact with parentheses, quotes, XML tags, and other common patterns that appear in text.

At this point, I don’t expect you guys understand what Text Objext is and I figure out that an example may do a better job, so here we go.

Let’s see a snippet from a html file: <p>Hello my friends</p>

Now we want to change the content of the tag ‘p’ from 'Hello my friends' to 'how are you', How can we do it?

You may say: That’s easy, using fH to go to the 'H', and then cfs. That’s not bad, but with VIM some better way.

Actually we could tell VIM we want to delete the content of the html tag by dit,and then enter insert mode to do the remaining job.

By dit, we mean: delete the content of the tag. Here it is a Text Object reprenting the content of the tag.

That’s pretty cool right?

In fact, VIM has way more Text Objects than 'it'. Follwing is a list of all the Text Objects you can use.

  1. a) or ab:A pair of (parentheses)
  2. i) or ib:Inside of (parentheses)
  3. a} or aB:A pair of {braces}
  4. i} or iB:Inside of {braces}
  5. a]:A pair of [brackets]
  6. i]:Inside of [brackets]
  7. a>:A pair of <lt;angle brackets>
  8. i>:Inside of <tangle brackets>
  9. a':A pair of ‘single quotes’
  10. i':Inside of ‘single quotes’
  11. a":A pair of “double quotes”
  12. i":Inside of “double quotes”
  13. a`:A pair of `backticks`
  14. i`:Inside of `backticks`
  15. at:A pair of <xml>tags</xml>
  16. it:Inside of <xml>tags</xml>
  17. aw:A word, including space
  18. iw:Inner word, not including space
  19. as:A sentence, including space
  20. is:Inner sentence, not including space
  21. ap:A paragraph, including space
  22. ip:Inner paragraph, not including space

Except the delete command, you can also combine text object with many other command like: yank commank y, the select command v etc. And I think it’s cumbersome to list all these commands out but if you want, you don’t have to spend a long time to figure out what these commands are.

And that’s all for the first post of the Learning VIM again series. Next post is coming soon.

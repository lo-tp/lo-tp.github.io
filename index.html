用vagrant搭建干净的hexo本地环境

近来觉得工作生活中都有许多值得记录的东西

遂决定开个blog

选用了相对简洁的hexo

本人有一定的系统洁癖

不希望在物理机上安装太多的东西

于是就有了这篇介绍hexo虚拟机环境搭建的文章

安装vagrant

如果是debian系的话，直接sudo apt-get install vagrant

当然如果想要最新版本，就到vagrant官网下载

添加basebox

这里我选用的是ubuntu 14.04

你如果不喜欢ubuntu，也可以从这个网址下载其他的发行版本

下载完成后运行

vagrant box add ubuntu14 xxxx.box

将下载好的basebox导入,其中ubuntu14是basebox导入本机后的名字

安装虚拟机

首先新建一个目录用于保存blog相关文件

进入这个目录

在这个目录下建立如下结构

▸ src/
initialize.sh
Vagrantfile

其中Vagrantfile文件内容为:

VAGRANTFILE_API_VERSION = "2"
Vagrant.configure(VAGRANTFILE_API_VERSION) do |config|
config.vm.box = "ubuntu14"
config.vm.provision :shell, :inline => "echo \"Asia/Shanghai\" | sudo tee /etc/timezone && dpkg-reconfigure --frontend noninteractive tzdata"
config.vm.provision :shell, path: "initialize.sh"
config.vm.synced_folder "src/", "/home/vagrant/src/",create:true
config.vm.network "private_network", ip: "192.168.50.7"
end

如果对上边这段代码有兴趣

请移步vagrant官网学习

需要说明的是

config.vm.provision :shell, :inline这一行用于将虚拟机的时区设置成中国的时区

initialize.sh的内容为:

#!/usr/bin/env bash

apt-get update
apt-get install screen -y

# For hexo
apt-get install nodejs -y
apt-get install npm -y
apt-get install git-core -y
npm install -g hexo 
ln -s /usr/bin/nodejs /usr/bin/node

一切无误，运行vagrant up

没有出现错误的话，虚拟机环境就搭建好了

初始化hexo

执行vagrant ssh连接到虚拟机

再运行如下两条命令

cd ~/src
hexo . init

之后根据个人需求_config.yml进行设置

具体设置可以参看hexo官网

测试hexo

hexo自带测试服务器,用于调试以及本地预览博客

如果你是按照我给出的vagrantfile安装的虚拟机的话

那么你只需要在虚拟机中运行如下两条命令

cd ~/src
hexo server -i 192.168.50.7

接着在物理机上打开浏览器，访问192.168.50.7:4000就能看见我们新搭建好的blog了

解决加载缓慢的问题

如果你身处天朝，那么可能会感觉加载的过程很慢

这是因为hexo里边引用了许多googleapi的静态资源

解决这个问题也很容易，只要将googleapi的静态资源引用转为其他cdn提供商的即可

这里假定读者使用的是chrome浏览器，其他浏览器原理相通

在访问blog之前，按f12打开开发工具

访问blog,审视开发工具中的network页

找出加载缓慢的资源

本例中加载缓慢资源地址为http//:ajax.googleapi.com/ajax/libs/jquery/2.0.3/jquery.min.js的js文件

使用grep命令找到相关文件

grep -rn '.' -e "googleapi"

本例中相关文件是themes/landscape/layout/_partial/after-footer.ejs

编辑这个文件

将<script src="//ajax.googleapi.com/ajax/libs/jquery/2.0.3/jquery.min.js"></script>改为
<script src="//ajax.useso.com/ajax/libs/jquery/2.0.3/jquery.min.js"></script>

从而将文件的引用就从google cdn转移到360的cdn上边

再试试，blog的访问是不是迅速了很多？
